# ezBookkeeping Two-Factor Recovery Codes Slice Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one-time recovery codes for users who enable TOTP two-factor authentication.

**Architecture:** Store recovery codes as BCrypt digests in a user-owned `TwoFactorRecoveryCode` table. Generate and display raw codes only once when 2FA is enabled or codes are regenerated. During the 2FA challenge, accept either a valid TOTP code or one unused recovery code, and mark recovery codes used immediately.

**Tech Stack:** Rails 8.1, Devise, Pundit, BCrypt, PostgreSQL, Minitest integration/model tests, Flowbite semantic classes.
