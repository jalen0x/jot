# ezBookkeeping Two-Factor Authentication Slice Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Rails-native TOTP two-factor authentication for Devise users.

**Architecture:** Store one enabled `TwoFactorAuthentication` record per user with a TOTP secret. Add a security settings page for setup/disable, and customize Devise sessions so password authentication for 2FA-enabled users creates a pending challenge instead of a signed-in session. Keep recovery codes, OIDC, persisted session lists, and app lock out of this slice.

**Tech Stack:** Rails 8.1, Devise, Pundit, ROTP, PostgreSQL, Minitest integration/model tests, Flowbite semantic classes.
